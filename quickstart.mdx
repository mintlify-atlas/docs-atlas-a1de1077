## Your First Debug Session

Let's walk through using Splat to debug a real Python error. This guide assumes you've already [installed Splat](/installation).

## Basic Usage

Splat's primary command is `splat squash` followed by your normal run command:

```bash
splat squash "python3 your_script.py"
```

<Note>
Use the same command you'd normally use to run your code, but wrap it in `splat squash`.
</Note>

## Example: Simple Syntax Error

Let's create a file with a common error and see how Splat fixes it.

<Steps>
  <Step title="Create a test file">
    Create a file called `test.py` with a syntax error:
    
    ```python test.py
    def greet(name):
        print("Hello, " + name
    
    greet("World")
    ```
    
    Notice the missing closing parenthesis on line 2.
  </Step>
  
  <Step title="Run Splat">
    Execute Splat with your test file:
    
    ```bash
    splat squash "python3 test.py"
    ```
  </Step>
  
  <Step title="Review the output">
    Splat will analyze the error and return structured information:
    
    ```json
    {
      "where": {
        "repository_path": "/Users/yourname/projects/myproject",
        "file_name": "test.py",
        "line_number": "2"
      },
      "what": {
        "error_type": "SyntaxError",
        "description": "Opening parenthesis in print statement was never closed"
      },
      "how": {
        "error_origination": "2",
        "suggested_code_solution": "    print(\"Hello, \" + name)\n"
      }
    }
    ```
    
    <Tip>
    Splat tells you exactly:
    - **Where**: Which file and line number
    - **What**: The type and description of the error  
    - **How**: The exact code to fix it
    </Tip>
  </Step>
</Steps>

## Deep Context with `-r` Flag

For more complex errors involving multiple files, use the `-r` (relational) flag to gather deep context.

### How `-r` Works

The `-r` flag enables relational context analysis:

<Accordion title="Technical Details: Relational Analysis">
  When you use `-r`, Splat:
  
  1. **Parses error stack**: Extracts all files mentioned in the traceback
     ```python
     # From relational.py:21
     collected_traceback_files = parse_error_stack(traceback)
     ```
  
  2. **Builds dependency graph**: Creates an adjacency list of all imports
     ```python
     # From relational.py:26
     graph = build_adjacency_list(collected_traceback_files, project_root)
     ```
  
  3. **Finds related files**: Recursively walks the graph to find all Nth-degree connections
     ```python
     # From relational.py:27-28
     all_related_files = get_nth_related_files(collected_traceback_files, graph)
     return traceback, error_information, run_mock_repopack(list(all_related_files))
     ```
  
  This provides the AI with complete context about your codebase structure.
</Accordion>

### Example: Multi-File Error

Create a project with multiple related files:

```python main.py
from utils import calculate

result = calculate(10, 0)
print(f"Result: {result}")
```

```python utils.py  
def calculate(a, b):
    return a / b  # Division by zero error
```

Run Splat with relational analysis:

```bash
splat squash -r "python3 main.py"
```

Splat will:
- Detect the error in `utils.py`
- Understand it was called from `main.py`  
- Provide context about both files
- Suggest a fix with error handling

<CodeGroup>
```json Splat Response
{
  "where": {
    "repository_path": "/path/to/project",
    "file_name": "utils.py",
    "line_number": "2"
  },

    "error_type": "ZeroDivisionError",
    "description": "Division by zero in calculate function"
  },
  "how": {
    "error_origination": "2",
    "suggested_code_solution": "    if b == 0:\n        raise ValueError('Cannot divide by zero')\n    return a / b\n"
  }
}
```

## Understanding the Flags

<Accordion title="No Flag: Basic Mode">
  Without any flags, Splat analyzes only the files mentioned in the error stack:
  
  ```bash
  splat squash "python3 app.py"
  ```
  
  **Best for**: Simple errors in single files or when you want fast results.
  
  ```python
  # From relational.py:30
  return traceback, error_information, run_mock_repopack(collected_traceback_files)
  ```
</Accordion>

<Accordion title="-r Flag: Relational Mode">
  With `-r`, Splat performs deep context analysis:
  
  ```bash
  splat squash -r "python3 app.py"
  ```
  
  **Best for**: Complex errors, import issues, or when you need comprehensive analysis.
  
  ```python
  # From relational.py:25-28
  if flag == '-r':
    graph = build_adjacency_list(collected_traceback_files, project_root)
    all_related_files = get_nth_related_files(collected_traceback_files, graph)
    return traceback, error_information, run_mock_repopack(list(all_related_files))
  ```
</Accordion>

## Command Structure

The general format for Splat commands:

```bash
splat squash <?-r> <!entrypoint>
```

- `squash`: The debug command
- `-r`: Optional flag for relational analysis
- `entrypoint`: Your normal run command (required)

### Valid Examples

<CodeGroup>
```bash Python Scripts
# Basic
splat squash "python3 app.py"

# With arguments
splat squash "python3 script.py --verbose"

# Relational mode
splat squash -r "python3 main.py"
```

```bash Python Modules
# Run module
splat squash "python3 -m mymodule"

# With relational analysis
splat squash -r "python3 -m mypackage.main"
```

```bash Other Commands
# pytest
splat squash "pytest tests/"

# Django
splat squash -r "python3 manage.py runserver"
```
</CodeGroup>

## Tips for Best Results

<Tip>
**Use `-r` when:**
- Error involves multiple files
- You have import or module errors
- The error context isn't obvious
- You want comprehensive analysis
</Tip>

<Tip>
**Skip `-r` when:**
- Simple syntax errors
- Single-file scripts
- You need quick results
- Error is already clear
</Tip>

<Warning>
**Current Limitations:**
- Splat works best when running from the project root directory
- Sub-module entrypoints may not work correctly if not in root
- Code fixes won't match your formatting configuration (use a formatter after)
</Warning>

## What Happens Behind the Scenes

When you run `splat squash`, here's what happens:

<Steps>
  <Step title="Execution & Capture">
    Splat runs your command in a subprocess and captures the error:
    
    ```python
    # From relational.py:14-19
    try:
      subprocess.run(entrypoint, capture_output=True, check=True, text=True)
    except subprocess.CalledProcessError as error:
      traceback: str = error.stderr if error.stderr else str(error)
      error_information: str = str(error)
    ```
  </Step>
  
  <Step title="Context Collection">
    Parses the traceback to find involved files:
    
    ```python
    # From utils/utils.py:47-74
    def parse_error_stack(error_info: str) -> List[str]:
      files = []
      file_pattern = re.compile(r'(?:File "([^"]+)"|(\S+\.py)\b)')
      for line in error_info.split('\n'):
        matches = file_pattern.findall(line)
        # Extract and validate file paths
      return list(dict.fromkeys(files))
    ```
  </Step>
  
  <Step title="AI Analysis">
    Sends context to Groq's LLM for analysis:
    
    ```python
    # From process/process.py:13-38  
    chat_completion = client.chat.completions.create(
      messages=[...],
      model="llama3-70b-8192",
      response_format={"type": "json_object"}
    )
    ```
  </Step>
  
  <Step title="Response">
    Returns structured JSON with error location, explanation, and fix.
  </Step>
</Steps>

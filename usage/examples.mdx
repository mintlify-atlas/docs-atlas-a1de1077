## Overview

This page provides practical examples of using Splat to debug various types of errors across different scenarios and languages.

## Quick Start Examples

### Simple Python Script

<Tabs>
  <Tab title="Error Code">
    ```python foo.py
    def greet(name):
        print(f"Hello, {name}"
    
    greet("World")
    ```
    
    Missing closing parenthesis on line 2.
  </Tab>
  
  <Tab title="Running Splat">
    ```bash
    $ splat squash "python3 foo.py"
    ```
    
    Output:
    ```
    🔎 Details about error
    ✅ We found the first instance of the error at line 2.
    ✅ The owner of the error: foo.py.
    ✅ The type of the error: SyntaxError.
    ✅ Isolated error message: '(' was never closed
    
    See suggested change? ... Yes / No
    ```
  </Tab>
  
  <Tab title="Solution">
    After selecting "Yes":
    
    ```json
    {
      "where": {
        "repository_path": "/home/user/project",
        "file_name": "foo.py",
        "line_number": "2"
      },
      "what": {
        "error_type": "SyntaxError",
        "description": "'(' was never closed on line 2"
      },
      "how": {
        "error_origination": "2",
        "suggested_code_solution": "print(f\"Hello, {name}\")"
      }
    }
    ```
    
    **Fixed code:**
    ```python
    print(f"Hello, {name}")
    ```
  </Tab>
</Tabs>

### Multi-File Import Error

<Tabs>
  <Tab title="Project Structure">
    ```plaintext
    myproject/
    ├── main.py
    ├── utils.py
    └── config.py
    ```
    
    ```python main.py
    from utils import process_data
    from config import API_KEY
    
    def run():
        result = process_data(API_KEY)
        print(result)
    
    if __name__ == "__main__":
        run()
    ```
    
    ```python utils.py
    def process_data(api_key):
        return api_key.lower()  # Error: API_KEY is None
    ```
    
    ```python config.py
    API_KEY = None  # Should be a string
    ```
  </Tab>
  
  <Tab title="Without -r Flag">
    ```bash
    $ splat squash "python3 main.py"
    ```
    
    **Context gathered:**
    - `main.py` (in stack trace)
    - `utils.py` (in stack trace)
    
    **Result:** May not have enough context to understand that `API_KEY` is incorrectly set in `config.py`.
  </Tab>
  
  <Tab title="With -r Flag">
    ```bash
    $ splat squash -r "python3 main.py"
    ```
    
    **Context gathered:**
    - `main.py` (in stack trace)
    - `utils.py` (in stack trace)
    - `config.py` (imported by main.py)
    
    **Result:** AI can see that `API_KEY = None` in `config.py` and suggest fixing the configuration.
    
    ```
    🔎 Details about error
    ✅ We found the first instance of the error at line 2.
    ✅ The owner of the error: utils.py.
    ✅ The type of the error: AttributeError.
    ✅ Isolated error message: 'NoneType' object has no attribute 'lower'
    
    Suggested fix: Set API_KEY to a string value in config.py
    ```
  </Tab>
</Tabs>

<Tip>
  For import-related errors, always use the `-r` flag to include all dependencies in the analysis.
</Tip>

## Common Error Scenarios

### Syntax Errors

<Tabs>
  <Tab title="Missing Parenthesis">
    ```python
    # Error
    print("Hello World"
    ```
    
    ```bash
    $ splat squash "python3 script.py"
    ```
    
    **Splat identifies:** Missing closing parenthesis  
    **Suggested fix:** Add `)` at end of line
  </Tab>
  
  <Tab title="Indentation Error">
    ```python
    def calculate(x, y):
    return x + y  # IndentationError
    ```
    
    ```bash
    $ splat squash "python3 script.py"
    ```
    
    **Splat identifies:** Incorrect indentation  
    **Suggested fix:** Add proper indentation (4 spaces)
  </Tab>
  
  <Tab title="Unclosed String">
    ```python
    message = "Hello World
    print(message)
    ```
    
    ```bash
    $ splat squash "python3 script.py"
    ```
    
    **Splat identifies:** Unterminated string literal  
    **Suggested fix:** Add closing quote
  </Tab>
</Tabs>

### Runtime Errors

<Tabs>
  <Tab title="NameError">
    ```python script.py
    def process():
        result = calculate_value()  # Function not defined
        return result
    
    process()
    ```
    
    ```bash
    $ splat squash "python3 script.py"
    ```
    
    **Output:**
    ```
    ✅ The type of the error: NameError.
    ✅ Isolated error message: name 'calculate_value' is not defined
    
    Suggested fix: Define calculate_value() or import it from another module
    ```
  </Tab>
  
  <Tab title="TypeError">
    ```python script.py
    def add_numbers(a, b):
        return a + b
    
    result = add_numbers(5, "10")  # Can't add int and str
    ```
    
    ```bash
    $ splat squash "python3 script.py"
    ```
    
    **Output:**
    ```
    ✅ The type of the error: TypeError.
    ✅ Isolated error message: unsupported operand type(s) for +: 'int' and 'str'
    
    Suggested fix: Convert string to int: add_numbers(5, int("10"))
    ```
  </Tab>
  
  <Tab title="AttributeError">
    ```python script.py
    class User:
        def __init__(self, name):
            self.name = name
    
    user = User("Alice")
    print(user.age)  # Attribute doesn't exist
    ```
    
    ```bash
    $ splat squash "python3 script.py"
    ```
    
    **Output:**
    ```
    ✅ The type of the error: AttributeError.
    ✅ Isolated error message: 'User' object has no attribute 'age'
    
    Suggested fix: Add age attribute in __init__ or use getattr with default
    ```
  </Tab>
</Tabs>

### Import Errors

<Tabs>
  <Tab title="Module Not Found">
    ```python app.py
    from utils import helper_function  # utils.py doesn't exist
    
    helper_function()
    ```
    
    ```bash
    $ splat squash "python3 app.py"
    ```
    
    **Output:**
    ```
    ✅ The type of the error: ModuleNotFoundError.
    ✅ Isolated error message: No module named 'utils'
    
    Suggested fix: Create utils.py or check import path
    ```
  </Tab>
  
  <Tab title="Circular Import">
    ```python a.py
    from b import function_b
    
    def function_a():
        return function_b()
    ```
    
    ```python b.py
    from a import function_a  # Circular dependency
    
    def function_b():
        return function_a()
    ```
    
    ```bash
    $ splat squash -r "python3 a.py"
    ```
    
    **Output:**
    ```
    ✅ The type of the error: ImportError.
    ✅ Isolated error message: cannot import name 'function_a' from partially initialized module 'a'
    
    Suggested fix: Restructure code to avoid circular imports or use late imports
    ```
  </Tab>
</Tabs>

## Advanced Scenarios

### Debugging Complex Frameworks

<Accordion title="Django ORM Query Error">
  **Scenario:** Complex query causing database error
  
  ```python models.py
  from django.db import models
  
  class Article(models.Model):
      title = models.CharField(max_length=200)
      published = models.BooleanField(default=False)
  ```
  
  ```python views.py
  from .models import Article
  
  def get_articles():
      # Error: Invalid field name
      return Article.objects.filter(publish=True)
  ```
  
  **Command:**
  ```bash
  splat squash -r "python manage.py runserver"
  ```
  
  **Analysis:**
  With `-r` flag, Splat includes both `views.py` and `models.py`, allowing the AI to see that the correct field name is `published`, not `publish`.
  
  **Suggested fix:**
  ```python
  return Article.objects.filter(published=True)
  ```
</Accordion>

<Accordion title="React Component State Error">
  **Scenario:** Incorrect state update
  
  ```jsx Counter.jsx
  import React, { useState } from 'react';
  
  function Counter() {
    const [count, setCount] = useState(0);
    
    const increment = () => {
      setCount(count + 1);
      setCount(count + 1);  // Doesn't work as expected
    };
    
    return <button onClick={increment}>{count}</button>;
  }
  ```
  
  **Command:**
  ```bash
  splat squash -r "npm start"
  ```
  
  **Analysis:**
  Identifies that state updates are batched and suggests using functional update form.
  
  **Suggested fix:**
  ```jsx
  const increment = () => {
    setCount(prev => prev + 1);
    setCount(prev => prev + 1);
  };
  ```
</Accordion>

### Async/Await Errors

<Tabs>
  <Tab title="Python AsyncIO">
    ```python async_app.py
    import asyncio
    
    async def fetch_data():
        await asyncio.sleep(1)
        return "data"
    
    # Error: Not using await
    result = fetch_data()
    print(result)  # Prints coroutine object, not "data"
    ```
    
    ```bash
    $ splat squash "python3 async_app.py"
    ```
    
    **Splat identifies:** Missing `await` keyword  
    **Suggested fix:**
    ```python
    result = await fetch_data()
    # Or:
    result = asyncio.run(fetch_data())
    ```
  </Tab>
  
  <Tab title="JavaScript Promises">
    ```javascript app.js
    function fetchData() {
        return new Promise(resolve => {
            setTimeout(() => resolve("data"), 1000);
        });
    }
    
    // Error: Not awaiting promise
    const result = fetchData();
    console.log(result.toUpperCase());  // TypeError
    ```
    
    ```bash
    $ splat squash "node app.js"
    ```
    
    **Splat identifies:** Missing `await` or `.then()`  
    **Suggested fix:**
    ```javascript
    const result = await fetchData();
    // Or:
    fetchData().then(result => console.log(result.toUpperCase()));
    ```
  </Tab>
</Tabs>

## Workflow Tips

### Iterative Debugging

<Steps>
  <Step title="First Pass - Quick Check">
    ```bash
    splat squash "python3 app.py"
    ```
    
    Start without flags for fast analysis of obvious errors.
  </Step>
  
  <Step title="Second Pass - Deep Analysis">
    ```bash
    splat squash -r "python3 app.py"
    ```
    
    If the error persists, add `-r` for comprehensive context.
  </Step>
  
  <Step title="Review and Apply">
    Carefully review the suggested fix before applying changes.  
    Use the interactive prompt to accept or reject suggestions.
  </Step>
  
  <Step title="Test">
    ```bash
    python3 app.py
    ```
    
    Run your application normally to verify the fix works.
  </Step>
</Steps>

## Troubleshooting

### No Error Detected

```bash
$ splat squash "python3 app.py"
# (Application runs successfully, no output from Splat)
```

If your application runs without errors, Splat exits silently. This is expected behavior.

### Too Much Context

If using `-r` includes too many files:

1. Start without `-r` to narrow down the issue
2. Add more entries to `.gitignore` to exclude unnecessary files
3. Ensure your project structure is modular

### AI Suggestions Not Helpful

If the AI's suggestions don't address your issue:

- Try adding `-r` flag for more context
- Check that all relevant files are being included
- Ensure error messages are clear and complete
- Consider if the error is in configuration or environment rather than code
